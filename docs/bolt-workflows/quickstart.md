# Bolt.new 快速开始指南

## 🚀 5分钟快速上手

### 步骤1：打开 Bolt.new
访问：https://bolt.new/

### 步骤2：设置项目
在聊天框中输入：

```
我要创建一个AI广告代投系统的前端项目

技术栈：
- Next.js 15 (App Router)
- TypeScript
- Tailwind CSS
- shadcn/ui 组件库
- Recharts 图表库

项目结构要求：
src/
├── app/          # Next.js 页面
├── components/   # 组件库
├── lib/         # 工具函数
├── types/       # 类型定义
└── public/      # 静态资源

主要功能模块：
- 日报管理 ✅
- 广告账户管理
- 对账系统
- 项目管理
- 财务管理
```

### 步骤3：开发第一个页面

在项目准备好后，输入：

```
帮我创建广告账户管理页面，基于我已有的日报管理页面的代码结构。

要求：
1. 使用相同的 PageTemplate 模板
2. 包含筛选、搜索、批量操作功能
3. 使用 DataTable 组件
4. 支持新建/编辑账户弹窗
5. 响应式设计
6. TypeScript 类型安全

请创建完整的页面代码
```

## 💡 Bolt.new 使用技巧

### 1. 提示词优化

#### 好的提示词示例
```
创建一个React组件，要求：

1. 技术要求：
   - 使用 TypeScript
   - 使用 Next.js 15 App Router
   - 使用 shadcn/ui 组件库
   - 支持响应式设计

2. 功能要求：
   - [详细描述功能]
   - [详细描述功能]

3. 代码质量：
   - 完整的类型定义
   - 错误边界处理
   - 加载状态处理
   - 无障碍访问

4. 设计要求：
   - 与现有设计系统保持一致
   - 支持主题切换（如果适用）
   - 优雅的加载和错误状态

5. 测试要求：
   - 包含基础测试
   - 包含使用示例

请创建完整的组件代码，包含所有必要文件。
```

#### 避免的提示词
```
❌ 创建一个按钮组件
✅ 创建一个Button组件，要求：

❌ 添加一些功能
✅ 添加以下具体功能：功能1、功能2
```

### 2. 迭代式开发

#### 第一轮 - 基础功能
```
创建基础版本的[组件名称]：
- 实现核心功能
- 基本样式
- 类型定义
```

#### 第二轮 - 功能增强
```
优化[组件名称]：
- 添加[新功能1]
- 改进[现有功能]
- 优化性能
- 增强用户体验
```

#### 第三轮 - 测试和文档
```
为[组件名称]创建：
- 单元测试
- 集成测试
- 使用文档
- 故障排除指南
```

### 3. 代码复用模式

#### 从现有代码中提取模式
```
基于我现有的日报管理组件，帮我创建[新模块]组件：

复用以下模式：
1. DataTable - 数据表格
2. FilterBar - 筛选栏
3. StatusBadge - 状态徽章
4. ActionButtons - 操作按钮

保持相同的设计风格和代码结构
```

#### 创建可复用的Hook
```
创建一个自定义Hook用于[功能描述]：

功能：
1. [具体功能1]
2. [具体功能2]
3. [具体功能3]

要求：
- 使用React Hooks最佳实践
- 类型安全
- 包含完整的测试
- 提供使用示例
```
